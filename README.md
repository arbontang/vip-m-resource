# vip-m-resource

一个用于归档和发布多个独立 HTML 页面的资源仓库。

## 项目简介

本仓库主要用于存放各类独立的 HTML 页面资源，涵盖工具页面、信息展示页、功能演示页等。所有页面均为静态 HTML 文件，不依赖后端服务，可直接通过浏览器打开或部署到任何静态托管平台。

仓库不针对任何特定域名、网站或服务，仅作为页面资源的中立存档与分发点。

## 目录结构

```
vip-m-resource/
├── pages/              # 存放所有独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：每个 HTML 文件为一个独立页面，文件名通常反映页面内容或功能。
- **assets/**：多个页面可共享的样式、脚本及图片资源，按类型分目录存放。

## 页面归档说明

- 所有页面均为纯前端实现，无后端依赖。
- 页面之间相互独立，互不干扰。
- 每个页面均可在浏览器中直接打开运行，或部署至任意静态服务器。
- 页面内容不涉及任何具体商业服务、推广信息或诱导性链接。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来添加或更新页面资源。
- 新增页面请放入 `pages/` 目录，并确保文件名简洁、无特殊字符。
- 公共资源请放入 `assets/` 对应子目录，避免重复引入。
- 不建议在页面内嵌入外部跟踪代码、广告或动态加载不可控资源。
- 本仓库仅作资源存档与共享，不对页面内容做额外保证。

## 使用方式

1. 克隆或下载本仓库。
2. 在浏览器中直接打开 `pages/` 下的任意 HTML 文件。
3. 或将整个 `pages/` 目录部署到 GitHub Pages、Netlify 等静态托管平台。
